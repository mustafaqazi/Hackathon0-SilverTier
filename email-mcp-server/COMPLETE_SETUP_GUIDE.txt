================================================================================
                       AVAILABLE TOOLS (5 Total)
================================================================================

1. send_email
   Send single email with HTML/text content

   Parameters:
     - to (required)       : Recipient email
     - subject (required)  : Email subject
     - html (optional)     : HTML content
     - text (optional)     : Plain text fallback
     - cc (optional)       : Array of CC recipients
     - bcc (optional)      : Array of BCC recipients

2. send_bulk_emails
   Send to multiple recipients at once

   Parameters:
     - recipients (required) : Array of email objects
                              Each with: to, subject, html, text, cc, bcc

3. send_email_from_template
   Use HTML template with variable substitution

   Parameters:
     - to (required)       : Recipient email
     - subject (required)  : Email subject
     - template (required) : Template name (e.g., "welcome")
     - data (optional)     : Object with variables

4. verify_email_config
   Check if email setup is working

   Parameters: (none)

   Returns:
     - verified: true/false
     - error message if failed
     - configuration details

5. list_templates
   List all available templates

   Parameters: (none)

   Returns:
     - count: number of templates
     - templates: array of template names
     - available: true/false

================================================================================
                      TROUBLESHOOTING QUICK REFERENCE
================================================================================

PROBLEM: "EMAIL_USER or EMAIL_PASS not set"
SOLUTION:
  1. Check .env file exists: ls .env
  2. Verify values are set: cat .env | grep EMAIL_
  3. No quotes around values
  4. Restart server: npm start

PROBLEM: "Invalid login"
SOLUTION:
  1. Use App Password, NOT Gmail password
  2. Generate new at: https://myaccount.google.com/apppasswords
  3. Remove spaces from password
  4. Update .env and restart

PROBLEM: "ENOTFOUND smtp.gmail.com"
SOLUTION:
  1. Check internet connection
  2. Check firewall (port 587 needed)
  3. Try disabling VPN
  4. Check ISP isn't blocking SMTP

PROBLEM: verify_email_config returns false
SOLUTION:
  1. Check credentials in .env
  2. Check 2-Factor Auth is enabled in Gmail
  3. Check App Password is correct
  4. Try generating new App Password

PROBLEM: Emails go to spam
SOLUTION:
  1. Add sender to Gmail contacts
  2. Mark email as "Not spam"
  3. Check SPF/DKIM configuration
  4. Ask recipient to whitelist sender
