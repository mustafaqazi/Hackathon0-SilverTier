# WhatsApp Watcher Setup Guide

## Overview
WhatsApp Watcher monitors unread WhatsApp Web messages for important keywords and creates action files in `vault/Needs_Action` for the Silver Tier AI Employee project.

## Features
✓ Monitors WhatsApp Web for unread messages
✓ Filters by keywords: urgent, asap, invoice, payment, help, sales, quote
✓ Creates markdown action files with YAML frontmatter
✓ Persistent browser session (no QR code scan each time)
✓ Checks every 60 seconds
✓ Tracks processed messages (no duplicates)
✓ Comprehensive logging (console + file)
✓ Exception handling for network errors, timeouts, etc.

## Configuration

### Monitored Keywords

Default keywords:
```python
KEYWORDS = ['urgent', 'asap', 'invoice', 'payment', 'help', 'sales', 'quote']
```

To modify, edit in `whatsapp_watcher.py`:

```python
KEYWORDS = ['urgent', 'asap', 'invoice', 'payment', 'help', 'sales', 'quote', 'custom']
```

### Check Interval

Default: 60 seconds

To change, edit in `run()` method:

```python
time.sleep(60)  # Change to desired seconds
```

Examples:
```python
time.sleep(30)    # Check every 30 seconds
time.sleep(120)   # Check every 2 minutes
time.sleep(300)   # Check every 5 minutes
```

### Headless Mode

Default: `headless=True` (browser runs hidden)

To see browser window (debugging):

Edit in `_setup_browser()`:

```python
self.browser = self.playwright.chromium.launch(headless=False)
```

## Troubleshooting

### Issue: "ModuleNotFoundError: No module named 'playwright'"

**Solution:**
```bash
pip install playwright
playwright install chromium
```

### Issue: Browser doesn't launch

**Solution:**
1. Check if Chromium is installed:
   ```bash
   playwright install chromium
   ```
2. Check if you have admin privileges
3. Check internet connection

### Issue: QR code scan timeout

**Symptoms:** "Timeout waiting for WhatsApp to load"

**Solution:**
1. Open console output URL manually if provided
2. Scan QR code with phone
3. Wait for connection (up to 2 minutes)

### Issue: No action files created

**Check:**
1. Are there unread messages in WhatsApp?
2. Do they contain monitored keywords?
3. Check logs: `vault/whatsapp_watcher_log.txt`
4. Try marking chat as unread and sending new message

### Issue: Session not saving

**Solution:**
1. Check folder permissions: `vault/whatsapp_session/`
2. Ensure folder is writable
3. Check available disk space

### Issue: Selector not found errors

**Cause:** WhatsApp Web UI changed

**Solution:**
1. WhatsApp frequently updates their UI
2. Check [WhatsApp Web GitHub](https://github.com/open-wa/wa-automate-nodejs) for selector updates
3. Update selectors in `whatsapp_watcher.py`

## FAQ

**Q: Will I need to scan QR code every time?**
A: No. First run only. Session is saved in `vault/whatsapp_session/`. Delete this folder to force re-authentication.

**Q: Can I run multiple instances?**
A: Not recommended. Session conflicts may occur. Use separate folders if needed.

**Q: Does it support WhatsApp groups?**
A: Yes. Groups appear as regular chats. Keywords trigger for group messages too.

**Q: Can I customize keywords?**
A: Yes. Edit `KEYWORDS` list in `whatsapp_watcher.py`.

**Q: What if WhatsApp blocks automation?**
A: WhatsApp Web doesn't officially support automation. Use at your own risk. WhatsApp may block accounts. Consider official WhatsApp Business API for production use.

**Q: How do I keep it running 24/7?**
A: Use process manager:
- Windows Task Scheduler
- Linux: `systemd` service or `screen`/`tmux`
- Mac: `launchd`
