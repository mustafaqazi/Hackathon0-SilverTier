# LinkedIn Watcher - Silver Tier Implementation

## Overview

**LinkedIn Watcher** is an automated script that monitors your LinkedIn messaging for sales-related conversations and creates action files for follow-up.

### Key Features

✅ **Automatic Sales Lead Detection** - Monitors LinkedIn for sales keywords
✅ **Persistent Sessions** - Logs in once, then stays authenticated
✅ **5-Minute Check Interval** - Lightweight monitoring with 300-second intervals
✅ **Headless Browser** - Runs efficiently in the background
✅ **Duplicate Prevention** - Tracks processed messages to avoid redundant files
✅ **Comprehensive Logging** - Console + file logging for troubleshooting
✅ **Markdown Action Files** - Creates structured files in vault/Needs_Action

---

## Installation & Setup

### Step 1: Install Playwright (if not already installed)

```bash
pip install playwright
playwright install
```

### Step 2: Run the Script

```bash
python scripts/linkedin_watcher.py
```

### Step 3: First-Time Login (First Run Only)

On the first run:
1. A browser window will open showing LinkedIn
2. **You must manually log in to LinkedIn** with your credentials
3. The script will wait up to 5 minutes for you to complete login
4. Once logged in, the session is saved automatically
5. Future runs will use the saved session - **no manual login needed!**

## How It Works

### 1. **Message Monitoring**
- Connects to `https://www.linkedin.com/messaging/`
- Scans for unread conversations in your inbox
- Extracts sender name and message preview

### 2. **Keyword Detection**
Monitors for these sales-related keywords:
```
lead, opportunity, sales, meeting, proposal,
connect, interested, quote, partnership, collaboration,
business, deal, contract, enquiry, request
```

### 3. **Action File Creation**
When a message contains matching keywords, creates a markdown file:

**Filename:** `LINKEDIN_[sender_name]_[timestamp].md`

**Location:** `vault/Needs_Action/`

**Content Example:**
```markdown
---
type: linkedin_message
from_sender: John Smith
received: 2026-02-19T15:30:45.123456
priority: medium
status: pending
keywords_matched: lead, opportunity
---

## Message Content

**From Sender:** John Smith
**Received:** 2026-02-19T15:30:45.123456
**Keywords Matched:** lead, opportunity

### Message
Hey, interested in discussing a partnership opportunity for our upcoming campaign...

## Suggested Actions
- [ ] Reply with sales pitch
- [ ] Schedule call
- [ ] Auto-post related content
- [ ] Add to CRM
- [ ] Forward to sales team

---
*Generated by LinkedIn Watcher on 2026-02-19T15:30:45.123456*
```

### 4. **Check Schedule**
- Checks every **300 seconds (5 minutes)**
- Runs in infinite loop until stopped with `Ctrl+C`
- Errors are logged but don't stop the script

## Troubleshooting

### ❌ "Timeout loading LinkedIn messaging page"
**Cause:** Browser couldn't connect to LinkedIn
**Solution:**
1. Check internet connection
2. Verify LinkedIn isn't blocking automated access
3. Try running again - sometimes it's a temporary network issue

### ❌ "LinkedIn login required"
**Cause:** Session expired or invalid
**Solution:**
1. Delete the session folder: `vault/linkedin_session/`
2. Run the script again and complete manual login
3. Session will be saved for future runs

### ❌ "No unread messages found"
This is normal! It means:
- There are no new unread messages
- All messages have been processed already
- Keep checking - the script will wait 5 minutes and try again

### ❌ "Failed to create action file"
**Cause:** File system permissions issue
**Solution:**
1. Check that `vault/Needs_Action/` folder is writable
2. Check disk space available
3. Ensure filename isn't too long (auto-limited to 50 chars)

### ❌ "No conversations found"
**Possible causes:**
1. Messaging page didn't fully load - script will retry in 5 minutes
2. LinkedIn changed its selectors (see "Selector Adjustment" below)
3. No messages in inbox - keep waiting

### 🔧 Selector Adjustment
If LinkedIn updates their page structure, selectors may need adjustment:

**Current selectors (line 87-90):**
```python
UNREAD_CONVERSATION_SELECTOR = '[data-unread="true"]'
CONVERSATION_ITEM_SELECTOR = '.msg-conversation-listitem'
MESSAGE_TEXT_SELECTOR = '.msg-s-message-list__content'
SENDER_NAME_SELECTOR = '.msg-s-msg-group__name'
```

To find correct selectors:
1. Open LinkedIn messaging in Chrome DevTools
2. Right-click on message element → Inspect
3. Find the selector in HTML
4. Update the constant in the script

## Integration with Other Watchers

LinkedIn Watcher follows the same pattern as:
- **Gmail Watcher** (`gmail_watcher.py`) - Email monitoring
- **WhatsApp Watcher** (`whatsapp_watcher.py`) - WhatsApp monitoring

All three can run simultaneously and feed into the same `vault/Needs_Action/` folder for unified task management.

### Running Multiple Watchers
```bash
# Terminal 1: Gmail Watcher
python scripts/gmail_watcher.py

# Terminal 2: LinkedIn Watcher
python scripts/linkedin_watcher.py

# Terminal 3: WhatsApp Watcher
python scripts/whatsapp_watcher.py
```

## Session Management

### Auto-Saving
- Session is automatically saved to `vault/linkedin_session/chrome_user_data/`
- No manual action needed
- Survives script restarts

### Manual Reset
To force a fresh login:
```bash
# Delete the session folder
rm -rf vault/linkedin_session/chrome_user_data/

# Run script again
python scripts/linkedin_watcher.py
```

## Security Notes
