================================================================================
WHAT HAPPENS NEXT
================================================================================

1. First Run:
   - Gmail Watcher initializes and loads settings
   - OAuth2 authentication flow opens in browser
   - You sign in to Gmail and grant "Read Gmail" permission
   - Token is saved automatically for future runs

2. Monitoring:
   - Checks Gmail every 120 seconds
   - Looks for: unread AND important (starred) emails
   - Creates .md files in vault/Needs_Action/ for new emails
   - Updates processed ID list to prevent duplicates
   - Logs all activity

3. Action Files Created:
   - Filename: EMAIL_[message_id].md
   - Location: vault/Needs_Action/
   - Content: YAML frontmatter + markdown with email details and action items
   - Each file is a task ready for processing

4. Continuous Operation:
   - Watcher runs forever
   - Automatically checks every 2 minutes
   - Press Ctrl+C to stop gracefully
   - All logs preserved in vault/gmail_watcher_log.txt

================================================================================
CONFIGURATION OPTIONS
================================================================================

To modify the watcher, edit gmail_watcher.py:

1. Check Interval (default: 120 seconds)
   Find: time.sleep(120)
   Change to desired seconds (e.g., 60, 300, 3600)

2. Email Query (default: is:unread is:important)
   Find: query = 'is:unread is:important'
   Change to custom query (e.g., 'is:unread from:boss@...')

3. Max Results (default: 10 per check)
   Find: maxResults=10
   Change to desired number

4. Log Level (default: INFO)
   Find: logger.setLevel(logging.INFO)
   Change to DEBUG, WARNING, or ERROR
