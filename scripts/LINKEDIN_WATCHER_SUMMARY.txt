================================================================================
KEY FEATURES IMPLEMENTED
================================================================================

✅ Sales Message Monitoring
   - Monitors: LinkedIn Messaging (https://www.linkedin.com/messaging/)
   - Target: Unread conversations with sales keywords
   - Frequency: Every 300 seconds (5 minutes)

✅ Keyword Detection (15 Keywords)
   Keywords monitored:
   - lead, opportunity, sales, meeting, proposal
   - connect, interested, quote, partnership, collaboration
   - business, deal, contract, enquiry, request

   All keywords are case-insensitive with word boundary checking

✅ Persistent Authentication
   - First run: Manual LinkedIn login (browser-based)
   - Future runs: Automatic session reuse
   - Session saved to: vault/linkedin_session/
   - No credential storage (browser-managed)

✅ Action File Creation
   - Format: Markdown with YAML frontmatter
   - Naming: LINKEDIN_[sender_name]_[timestamp].md
   - Location: vault/Needs_Action/
   - Includes:
     * Sender name
     * Message preview
     * Matched keywords
     * Suggested actions
     * Metadata (priority, status, received time)

✅ Duplicate Prevention
   - Tracks processed messages in JSON file
   - Prevents creating duplicate action files
   - Survives script restarts
   - File: .linkedin_processed_messages.json

✅ Comprehensive Logging
   - Console output: Real-time status updates
   - File logging: vault/linkedin_watcher_log.txt
   - Formatted: Timestamp + Level + Message
   - Tracks: Initialization, checks, results, errors

✅ Error Handling
   - Timeout handling (LinkedIn page load)
   - Login detection and waiting
   - Missing selector fallbacks
   - Network error recovery
   - Graceful shutdown (Ctrl+C)

✅ Headless Browser
   - Technology: Playwright (Chromium)
   - Mode: Headless (efficient background running)
   - Session: Persistent across restarts
   - Detection avoidance: Stealth scripts injected

================================================================================
TECHNICAL ARCHITECTURE
================================================================================

Class: LinkedInWatcher
   __init__()                      - Initialize paths, logging, config
   _load_processed_messages()      - Load tracking cache
   _save_processed_messages()      - Save tracking cache
   _setup_browser()                - Launch Playwright with session
   _close_browser()                - Cleanup resources
   _navigate_to_linkedin()         - Load messaging page, handle login
   _extract_unread_messages()      - Parse conversation list
   _check_keywords()               - Detect sales keywords
   _sanitize_filename()            - Safe filename generation
   create_action_file()            - Generate markdown action file
   check_for_updates()             - Main check logic
   run()                           - Infinite loop runner

Data Flow:
1. Start → Initialize paths & logging
2. Setup browser with persistent session
3. Navigate to LinkedIn messaging
4. Wait for first manual login (if needed)
5. Loop:
   a. Extract unread conversations
   b. Check message text for keywords
   c. Create action files for matches
   d. Sleep 5 minutes
6. Graceful shutdown on Ctrl+C

CONFIGURATION OPTIONS
================================================================================

Easily Customizable (Edit linkedin_watcher.py):

1. Check Interval (Default: 300 seconds / 5 minutes)
   Line 546: time.sleep(300)
   Change to: 60 (1 min), 120 (2 min), 600 (10 min), etc.

2. Sales Keywords (Default: 15 keywords)
   Lines 79-81: KEYWORDS = [...]
   Add/remove keywords as needed
   Example: 'demo', 'budget', 'timeline', 'approve'

3. Priority Level (Default: medium)
   Line 465: priority: medium
   Change to: 'high' or 'low'

4. Browser Mode (Default: headless=True)
   Line 197: headless=True
   Set to False to see browser while running

5. Check Interval Message (Default: 300 seconds)
   Line 547: "Waiting 300 seconds..."
   Update to match your interval

================================================================================
USAGE - QUICK START
================================================================================

1. Prerequisites:
   pip install playwright
   playwright install chromium

2. First Run (Requires Manual Login):
   python scripts/linkedin_watcher.py
   - Browser will open
   - Login to LinkedIn manually
   - Session is saved

3. Subsequent Runs (Automatic):
   python scripts/linkedin_watcher.py
   - Browser opens with saved session
   - No manual login needed
   - Starts monitoring immediately

4. Monitor Action Files:
   Check vault/Needs_Action/ for generated files
   Each file: LINKEDIN_[name]_[timestamp].md

5. Stop Watcher:
   Press Ctrl+C in terminal

================================================================================
TROUBLESHOOTING QUICK REFERENCE
================================================================================

Error: "Playwright not installed"
→ Solution: pip install playwright && playwright install chromium

Error: "Timeout loading LinkedIn"
→ Solution: Check internet, try again, or check LinkedIn's robot detection

Error: "Login required" on 2nd run
→ Solution: Delete vault/linkedin_session/ and login again

Error: "No unread messages found"
→ This is normal! No new messages or all already processed

Error: "Failed to create action file"
→ Solution: Check vault/Needs_Action/ permissions, check disk space

Error: "browser window didn't open"
→ Solution: Set headless=False to debug, check system resources

For more details:
→ See: LINKEDIN_WATCHER_SETUP.md
